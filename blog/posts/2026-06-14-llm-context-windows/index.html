The Long Memory: How LLMs Learned to Read Entire Libraries

In 2017, the original Transformer could hold roughly 512 tokens in its head at once — a few paragraphs. By 2024, Gemini 1.5 Pro was processing 2 million tokens, enough to ingest a stack of novels in a single pass. That is a 4,000× expansion in less than a decade.

This post traces every meaningful step on that journey. Not just what changed, but why each technique existed, what wall it hit, and what came next to break through it.

The Baseline: Transformers and the O(n²) Wall (2017, 512 tokens)

The original "Attention Is All You Need" (Vaswani et al., 2017) introduced Multi-Head Attention (MHA) — the mechanism that lets every token attend to every other token in the sequence.

How Multi-Head Attention Works

For a sequence of length \(n\) with model dimension \(d\), MHA projects inputs into \(h\) parallel heads, each computing:

\[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]

Each head independently computes attention, then outputs are concatenated and projected:

\[\text{MHA}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)W^O\]

The richness of MHA comes from different heads learning different relational patterns — syntactic structure, coreference, semantic similarity — simultaneously.

The O(n²) Wall

The softmax(QKᵀ) step computes an \(n \times n\) attention matrix. This is the fundamental bottleneck:

  • Compute: \(O(n^2 \cdot d)\) FLOPs
  • Memory: \(O(n^2)\) — the attention matrix must be materialized in GPU SRAM

At 512 tokens, that matrix is 512×512 = 262K entries. Manageable. At 100K tokens, it's 10 billion entries — impossible to hold in any GPU's high-bandwidth memory. This quadratic scaling is the wall every subsequent technique tries to break, route around, or chip away at.

Positional Encodings: Sinusoidal vs. Learned (2017–2018)

Attention is permutation-invariant by design — the matrix QKᵀ doesn't know which token came first. Position must be injected separately.

Sinusoidal Encodings (Vaswani et al., 2017)

The original Transformer adds fixed sinusoidal signals to token embeddings before attention:

\[PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right), \quad PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)\]

Advantage: Extrapolates beyond training length in theory.
Limitation: In practice, models trained on 512-token sequences perform poorly on 1024-token inference because attention distributions are calibrated to that positional scale.

Learned Positional Embeddings (BERT, GPT-2, 2018–2019)

BERT and GPT-2 replaced sinusoidal signals with learned embedding tables — one trainable vector per position. More expressive but strictly cap generalization at the training length. Position 513 has no learned embedding if training only saw 512.

The hard lesson: Both approaches made context length a hyperparameter baked into training. To extend it, you needed to retrain — or find a smarter encoding.

Sparse Attention: Longformer and BigBird (2020, 4K–16K tokens)

Rather than letting every token attend to every other, what if we restrict attention patterns to only the connections that matter most?

Longformer (Beltagy et al., 2020)

Longformer introduced a combination of three attention patterns:

  1. Sliding Window (Local) Attention: Each token attends to \(w\) neighbors on each side. Cost: \(O(n \cdot w)\) — linear in sequence length.
  2. Dilated Sliding Window: Gaps between attended positions, increasing receptive field without more compute.
  3. Global Attention: A small set of special tokens (e.g., [CLS]) attend to everything. This propagates global context through the network.

Total complexity: \(O(n \cdot w + n_g \cdot n)\) where \(n_g \ll n\) is the number of global tokens. Longformer achieved 4,096 tokens as a practical context.

BigBird (Zaheer et al., 2020)

BigBird combined local window attention, global tokens, and random attention: each token attends to \(r\) randomly sampled tokens, ensuring theoretical universal approximation. BigBird proved that \(O(n)\) sparse attention can approximate full \(O(n^2)\) attention with bounded error.

Limitation shared by both: Sparse patterns require custom CUDA kernels, and for tasks needing any two distant tokens to interact directly, information must flow through many layers, potentially diluting signal.

FlashAttention: Making O(n²) Fast Through Hardware Awareness (2022)

The key insight: the O(n²) wall isn't purely algorithmic — it's a memory bandwidth problem.

The Tiling Trick

Standard attention materializes the full \(n \times n\) attention matrix in High-Bandwidth Memory (HBM). GPU HBM is large but slow (~2 TB/s for A100). The GPU's SRAM is tiny (~20 MB) but fast (~19 TB/s).

FlashAttention (Dao et al., 2022) tiles the computation into blocks that fit in SRAM:


for block i in Q:
    for block j in K, V:
        load Q_i, K_j, V_j into SRAM
        compute local attention scores
        accumulate into output O_i using online softmax
        write only O_i back to HBM  # full matrix never written

Online Softmax

Softmax requires seeing all scores to normalize. FlashAttention uses numerically stable online softmax — maintaining running max and sum across blocks:

\[m_i = \max(m_{i-1}, \text{rowmax}(S_{ij})), \quad \ell_i = e^{m_{i-1}-m_i} \ell_{i-1} + \text{rowsum}(e^{S_{ij}-m_i})\]

Result: IO complexity drops from \(O(n^2 d)\) HBM reads/writes to \(O(n^2 d / M)\) where \(M\) is SRAM size. In practice: 2–4× wall-clock speedup and 5–20× memory reduction on long sequences. FlashAttention v2 (2023) achieved ~72% of A100 peak FLOPs on attention kernels.

Key insight: FlashAttention doesn't change algorithmic complexity — it's still O(n²) in FLOPs. It makes the constant factor dramatically smaller by respecting the GPU memory hierarchy.

RoPE: Rotary Position Embeddings (2021)

Su et al. (2021) asked: what if position wasn't added to embeddings, but encoded as a rotation applied during attention?

The Mechanism

For query vector \(q_m\) at position \(m\) and key vector \(k_n\) at position \(n\), RoPE applies a rotation matrix before the dot product:

\[q_m^T k_n = (R_{\Theta,m} q)^T (R_{\Theta,n} k) = q^T R_{\Theta,n-m} k\]

The dot product now depends only on the relative position \((n - m)\), not absolute positions. In practice, RoPE rotates pairs of dimensions:

\[\text{RoPE}(x, m) = \begin{pmatrix} x_1 \cos m\theta_1 - x_2 \sin m\theta_1 \\ x_1 \sin m\theta_1 + x_2 \cos m\theta_1 \\ \vdots \end{pmatrix}, \quad \theta_i = 10000^{-2i/d}\]

Why RoPE Matters for Long Context

  1. No learned table: No hard length cap from embedding table size
  2. Relative distances: Models trained on short sequences generalize better to longer ones
  3. Widely adopted: LLaMA, Mistral, Falcon, Gemma, Qwen, and virtually all modern open-source models use RoPE

Limitation: When inference sequences exceed training length, rotation angles produce values outside the training distribution. Direct extrapolation degrades rapidly.

ALiBi: Attention with Linear Biases (2022)

Press et al. (2022) added a learned linear penalty to attention scores instead of positional embeddings:

\[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} - m \cdot |i - j|\right)V\]

The scalar \(m\) is a head-specific slope (fixed from a geometric sequence). Tokens far apart get penalized with larger negative biases, making attention prefer local context.

Advantage: Excellent length generalization — models trained on 1K tokens perform surprisingly well at 2K during inference.
Disadvantage: The hard bias toward local attention can hurt tasks requiring long-range exact recall.

YaRN and NTK-Aware Scaling: Extending RoPE Beyond Training Length (2023)

The key insight from NTK theory: RoPE's high-frequency dimensions cycle rapidly and generalize well; low-frequency dimensions are effectively learned and don't extrapolate. The fix is to scale frequencies to spread the training distribution across longer sequences.

Position Interpolation (Chen et al., 2023)

The simplest fix: divide all positions by a scaling factor \(s = L'/L\):

\[\text{PI: } m \rightarrow m / s\]

This compresses positions back into the training distribution but reduces resolution — nearby tokens lose fine-grained positional sensitivity.

NTK-Aware Scaling (bloc97, 2023)

Scale the base of the frequency formula rather than positions:

\[\theta_i' = (10000 \cdot s)^{-2i/d}\]

This stretches low-frequency dimensions while preserving high-frequency ones. Better length generalization with less perplexity degradation.

YaRN (Peng et al., 2023)

YaRN combines interpolation, NTK-aware scaling, and a temperature adjustment to the attention entropy:

\[\text{YaRN: softmax}\left(\frac{QK^T}{\sqrt{d_k} \cdot t}\right)\]

where \(t\) counteracts the flattening of attention distributions at longer sequences. YaRN allowed LLaMA-2 (trained on 4K) to effectively handle 128K tokens with minimal fine-tuning.

Grouped Query Attention (GQA): Memory Efficiency at Scale (2023)

As context grows, the KV cache becomes a dominant memory cost. For batch size \(B\), sequence length \(n\), with \(h\) heads and dimension \(d\):

\[\text{KV cache size} = 2 \cdot B \cdot n \cdot h \cdot d \cdot \text{bytes\_per\_param}\]

At 100K tokens, B=1, h=32, d=128: that's ~3.2 GB in fp16 for keys+values alone. Multi-Query Attention (Shazeer, 2019) used a single KV head for all queries — 4× reduction but quality loss. GQA (Ainslie et al., 2023) found the sweet spot: group query heads into \(g\) groups, each sharing one KV head.

LLaMA-3, Mistral, Gemma-2, and Qwen2 all use GQA with \(g=8\) (8 KV heads for 32+ query heads): a 4× KV cache reduction enabling longer effective context with the same GPU memory.

Sliding Window Attention: Mistral's Local Attention (2023)

Mistral 7B (Jiang et al., 2023) incorporated Sliding Window Attention (SWA) directly into a production model. Each token attends only to the \(w\) preceding tokens (window size \(w = 4096\)). Through \(L\) layers, the effective receptive field grows as \(w \times L\) — Mistral's 32-layer model had an effective context of ~131K tokens despite a 4K window per layer.

Combined with a rolling buffer KV cache, Mistral discards KV entries older than the window, keeping memory constant regardless of total sequence length processed. Complexity: \(O(n \cdot w)\).

Claude's 200K context was notable for high recall accuracy at long range, suggesting Anthropic combined architectural tricks with careful training on long-document data.

Ring Attention: Distributed Context Across Devices (2023)

Li et al. (2023) introduced Ring Attention to enable sequences too long for a single device by distributing attention across a ring of GPUs:


Device 1: holds Q1, K1, V1
Device 2: holds Q2, K2, V2
...
Device N: holds QN, KN, VN

Each step: compute local attention block
           pass KV to next device in ring (all-reduce-like)
           accumulate output using online softmax

Since KV blocks are passed in a ring while local QKV computation happens simultaneously, communication is overlapped with computation — effectively hiding inter-device latency. Ring Attention is what enables Gemini 1.5's million-token context: the sequence is spread across a TPU pod.

Mixture of Experts: Context + Efficiency (2022–2024)

Sparse MoE doesn't directly extend context windows, but enables the model capacity needed to use long context without proportional compute increases. For each token, a router selects \(k\) of \(E\) experts to activate:

\[y = \sum_{i=1}^{E} G(x)_i \cdot E_i(x), \quad G(x) = \text{Top-}k(\text{softmax}(Wx))\]

Mixtral 8×7B (2023): 8 experts, 2 activated per token. Total parameters: 46B. Active parameters per token: ~13B. You get 46B-quality representations at 13B inference cost. This efficiency headroom is deployed toward longer context. Gemini 1.5 (2024) combines MoE with Ring Attention to reach 1M tokens.

Gemini 1.5 Pro: 1M–2M Tokens (2024)

Google's Gemini 1.5 Pro pushed context to 1 million tokens (with 2M in experimental mode), enabling full codebase ingestion, hour-long video understanding, and multi-document legal analysis.

Technical stack (inferred from paper + architecture trends):

  • MoE architecture for parameter efficiency
  • Ring Attention for distributed sequence parallelism across TPU pods
  • RoPE + LongRoPE for positional encoding at extreme lengths
  • Multi-modal token compression for non-text modalities

Benchmark results showed >99% recall accuracy on needle-in-a-haystack retrieval at 1M tokens — context wasn't just available, it was actually used.

LongRoPE: Pushing RoPE to Extreme Lengths (2024)

Ding et al. (2024) from Microsoft observed that standard RoPE extension methods applied uniform scaling — but different RoPE dimensions have different sensitivities. LongRoPE uses non-uniform scaling: each dimension \(i\) gets its own scaling factor \(\lambda_i\), determined by evolutionary search:

\[\theta_i' = \theta_i \cdot \lambda_i\]

Additionally, LongRoPE uses a two-phase training strategy: fine-tune at 256K context, then scale up to the target 2M context via a final non-uniform interpolation pass. Microsoft's Phi-3 Mini (3.8B parameters) was extended to 128K context using LongRoPE, achieving better perplexity than YaRN or NTK-aware scaling with the same fine-tuning budget.

Mamba and SSMs: Breaking the O(n²) Paradigm (2023–2024)

All the techniques above fight the \(O(n^2)\) cost of attention. Structured State Space Models (SSMs) sidestep it entirely.

The State Space Formulation

An SSM maps input \(u(t)\) to output \(y(t)\) through a latent state \(x(t)\):

\[\dot{x}(t) = Ax(t) + Bu(t), \quad y(t) = Cx(t) + Du(t)\]

In discrete form, this becomes a linear recurrence — computable in \(O(n)\) sequentially, or \(O(n \log n)\) in parallel using the parallel scan algorithm.

Mamba (Gu & Dao, 2023)

The key limitation of earlier SSMs (S4, H3) was fixed \(A, B, C\) matrices — the model couldn't selectively focus on relevant inputs. Mamba introduced selective state spaces: parameters become functions of the input:

\[B_t = \text{Linear}(u_t), \quad C_t = \text{Linear}(u_t), \quad \Delta_t = \text{Softplus}(\text{Linear}(u_t))\]

The \(\Delta_t\) (discretization step) controls how much each timestep updates the state. For relevant tokens, \(\Delta_t\) is large (state absorbs the input). For irrelevant tokens, \(\Delta_t\) is small (state carries forward unchanged). This is selective attention implemented as a state update, not a dot product.

Complexity: \(O(n \cdot d_{state})\) — linear in sequence length. Memory is \(O(1)\) during inference (unlike KV cache which grows as \(O(n)\)).

Tradeoff: SSMs excel at compression and streaming inference, but struggle with exact recall of arbitrary past tokens. If the relevant token was overwritten in the state, it's gone. Attention can always retrieve a specific past token; Mamba cannot guarantee this.

Hybrid Architectures (2024)

The field has converged on hybrid models — interleaving attention layers (for precise retrieval) with SSM layers (for efficient summarization):

  • Jamba (AI21, 2024): Mamba + Transformer layers + MoE
  • Zamba (Zyphra, 2024): 6:1 Mamba-to-attention ratio
  • Falcon Mamba (TII, 2024): Pure SSM at 7B scale

The bet: use attention sparingly for retrieval-critical layers, SSMs for cheap long-range compression.

Why This All Matters

Context length isn't just a benchmark number. It fundamentally changes what you can ask a language model to do:

  • 512 tokens: Answer a question about a paragraph
  • 4K tokens: Summarize a short article, hold a multi-turn conversation
  • 32K tokens: Analyze a research paper end-to-end
  • 128K tokens: Audit a codebase, process a legal contract
  • 1M tokens: Ingest an entire software project, analyze hours of video

The progression wasn't linear. It required breakthroughs in hardware-aware algorithms (FlashAttention), positional encoding theory (RoPE/YaRN), memory efficiency (GQA), distributed systems (Ring Attention), and entirely new computational paradigms (SSMs).

The next frontier isn't just longer — it's smarter memory. Models that learn which parts of a long context to retrieve, compress, or discard. That's the problem Mamba-style hybrids, compressed KV cache methods (like H2O and Scissorhands), and memory-augmented architectures are beginning to tackle.
